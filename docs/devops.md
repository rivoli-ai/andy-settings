# DevOps and Containerization

## Overview

Andy Settings follows the same operational patterns as Andy Containers and Andy Code Index:

- Docker-first local orchestration
- Multi-stage Dockerfile (Node + .NET + runtime)
- Docker Compose with PostgreSQL
- Corporate CA certificate support
- Self-signed dev certificates
- MkDocs Material for documentation

## Local Development Modes

### Mode 1 -- Native Local Dev

Use for rapid development on macOS.

```bash
# Backend
dotnet run --project src/Andy.Settings.Api

# Frontend
cd client && npm install && npm start

# CLI
dotnet run --project tools/Andy.Settings.Cli -- --help
```

- SQLite database (set `Database__Provider=Sqlite`)
- Angular dev server with proxy to backend
- No Docker required

### Mode 2 -- Docker Compose Dev

Use for end-to-end containerized validation.

```bash
docker compose up --build
```

- API container with PostgreSQL
- Angular SPA served from wwwroot
- Corporate CA cert support via `certs/` directory

## Dockerfile

Multi-stage build matching the Andy ecosystem pattern:

1. **Node build stage** -- Angular SPA compilation (Node 22 Alpine)
2. **.NET build stage** -- Restore, build, publish (.NET SDK 8.0)
3. **Runtime stage** -- Minimal aspnet:8.0 image with non-root user

Key features:

- Corporate CA certificate injection in all stages
- Self-signed dev certificate generation
- Non-root `andysettings` user
- Data Protection keys volume
- Health check support

## Docker Compose Services

| Service | Image | Host Port | Container Port | Description |
|---------|-------|-----------|----------------|-------------|
| `postgres` | postgres:16-alpine | 7438 | 5432 | Database |
| `api` | (built from Dockerfile) | 7300 (HTTPS), 7301 (HTTP) | 8443, 8080 | API + SPA |

Environment variables for the API container:

```yaml
ASPNETCORE_ENVIRONMENT: Development
ASPNETCORE_URLS: https://+:8443;http://+:8080
ConnectionStrings__DefaultConnection: Host=postgres;Port=5432;...
Database__Provider: PostgreSql
AndyAuth__Authority: https://host.docker.internal:5001
AndyAuth__Audience: urn:andy-settings-api
Rbac__ApiBaseUrl: https://host.docker.internal:5003
Rbac__ApplicationCode: settings
```

## Conductor Embedded Mode

When running inside Conductor, no Docker is needed:

- Conductor's `ServiceOrchestrator` launches the .NET process
- Port 9111, prefix `/settings`
- SQLite at `~/Library/Application Support/ai.rivoli.conductor/db/andy-settings.sqlite`
- `Database__Provider=Sqlite` environment variable

## Certificates

The `certs/` directory holds corporate CA certificates for environments behind TLS-inspecting proxies.

- Place `.crt`, `.pem`, or `.cer` files in `certs/`
- Files are automatically trusted during Docker build
- Runtime-mounted certs are trusted at container startup
- Local dev: self-signed certificates generated by Angular CLI and Kestrel

## Database Operations

### Migrations

```bash
# Generate migration
dotnet ef migrations add MigrationName \
  --project src/Andy.Settings.Infrastructure \
  --startup-project src/Andy.Settings.Api

# Apply migration
dotnet ef database update \
  --project src/Andy.Settings.Infrastructure \
  --startup-project src/Andy.Settings.Api
```

Development mode: auto-migration on startup.

### Provider Switching

Set `Database:Provider` in appsettings or `Database__Provider` environment variable:

- `PostgreSql` (default in Docker Compose)
- `Sqlite` (default in Conductor / native local dev)

SQLite default path: `~/.andy-settings/andy-settings.sqlite`

## Health Checks

- PostgreSQL: `pg_isready -U andy_settings -d andy_settings`
- API: `curl -fk https://localhost:8443/health`
- Conductor: `ServiceHealthMonitor` polls `/health` (10s timeout, 30s cold-start grace)

## CI Workflow

Suggested GitHub Actions stages:

1. **Build & Lint** -- restore, build solution, build Angular client
2. **Test** -- unit tests, integration tests, Angular tests
3. **Container Build** -- build Docker image, smoke health check
4. **Docs** -- build MkDocs, deploy to GitHub Pages
